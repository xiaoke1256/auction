# 拍卖系统 Demo（仅可演示界面）

本目录为拍卖系统的**纯前端演示**，不连接数据库，所有数据为浏览器内的模拟数据。刷新页面会恢复初始状态。已实现**登录与权限**：管理人员与竞拍人分端登录，按角色展示功能入口并做页面级校验。

## 登录与权限（演示）

- **管理人员**：由超级管理员授权，不开放自助注册。  
  - 演示账号：**admin / admin**（超级管理员）、**manager / 123**（普通管理员）  
  - 登录后可见：委托人及拍买管理、拍品公示、竞价（监控）、结果、成交书、退还保证金  
- **竞拍人**：在平台**自助注册**后登录。  
  - 演示账号：**13900002001 / 123**、**13600002002 / 123**  
  - 注册入口：首页「竞拍人注册」或 [signup.html](signup.html)  
  - 登录入口：首页「竞拍人登录」或 [login-bidder.html](login-bidder.html)  
  - 登录后可见：拍品公示、竞价报名、竞价（出价）  
- **未登录**：仅可浏览**拍品公示**；点击「竞价报名」「竞价」会跳转竞拍人登录页，登录后返回原页面。  
- 管理员与竞拍人登录为**独立页面**：[login-admin.html](login-admin.html)、[login-bidder.html](login-bidder.html)；[login.html](login.html) 仅提供入口选择。

## 如何运行

### 方式一：直接打开（推荐）

用浏览器直接打开根目录下的 `index.html` 即可：

- 双击 `demo/index.html`，或  
- 在文件管理器中右键 → 用浏览器打开

注意：若从 `file://` 打开，部分浏览器可能对本地脚本有限制；若遇问题请用方式二。

### 方式二：本地静态服务

在项目根目录执行：

```bash
npx serve demo
```

然后访问终端提示的地址（一般为 `http://localhost:3000`）。

或进入 `demo` 目录后：

```bash
cd demo
npx serve .
```

## 功能入口

从 `index.html` 可进入以下 7 个模块：

| 模块 | 页面 | 说明 |
|------|------|------|
| 1. 委托人及拍买管理 | `pages/consignor-lots.html` | 委托人管理、拍品管理、拍卖管理（含拍卖委托书签署） |
| 2. 拍品公示 | `pages/public-lots.html` | 公示列表、详情、报名入口 |
| 3. 竞价报名 | `pages/register.html` | 选择拍品、保证金（底价×20%），信息由竞拍人登记处带入；报名后待管理员审核，可查看「我的报名」状态 |
| 报名审核 | `pages/approve-register.html` | 管理员审核：待审核/已通过/已拒绝；通过后竞拍人取得该拍品竞拍资格 |
| 4. 竞价 | `pages/bidding.html` | 选择拍卖中拍品、当前最高价、出价记录、提交出价 |
| 5. 拍卖结果汇总及发布 | `pages/results.html` | 结果列表、导出 CSV |
| 6. 签订成交书 | `pages/contract.html` | 成交书内容展示、确认签署 |
| 7. 退还保证金 | `pages/refund.html` | 待退款/已退款列表、确认退款 |

## 数据说明

- 数据定义在 `js/mock-data.js`，包含委托人、拍品、竞拍人、保证金、出价、结果、成交书等。
- 在页面上的「保存」「确认」「出价」「登记」等操作仅修改当前页或内存中的 JS 数据，**刷新后恢复为初始 mock 数据**。
- 支付与退款按需求为线下进行，演示中通过「模拟保证金到账」「确认退款」等按钮完成状态变更。
